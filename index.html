Intro to Comp Sys (CMU 15213/15513) Labs
Summer 2022

The labs of Intro to Computer System (15513) are very high quality. I've learned a lot about how different systems work in the computer through the simulations of these labs. The following are the four largest and complicated labs of 15513:

Cache Lab: Understanding Cache Memories. The lab consists of three parts: 1) write several traces to test the behavior of a cache simulator, 2) write a small C program that simulates the behavior of a hardware cache memory, and 3) optimize a matrix transpose function, with the goal of minimizing the number of cache misses. Here, I used the idea of blocking.

Malloc Lab: Writing a Dynamic Storage Allocator. I wrote a correct, efficient, and fast dynamic memory allocator (similar to malloc functionality in C) which consist of the malloc, free, realloc, and calloc functions. Here, I used the idea of segmental explicit free list (with anonymous struct and anonymous union) to maintain the free chuncks, coalescing neibouring free chuncks, first fit, eliminated footer of allocated chuncks by adding a flag bit in the header to indicate previous allocation status, and maintianed a seperate list for mini blocks (16KB).

Shell Lab: Writing Your Own Linux Shell. The purpose of this assignment is to utilize the concepts of process control and signalling by writing a simple Linux shell program, tsh (tiny shell), that supports a simple form of job control and I/O redirection. Tsh is an interactive command-line interpreter that runs programs on behalf of the user. It repeatedly prints a prompt, waits for a command line on stdin, and then carries out some action, as directed by the contents of the command line. Built-in commands run within the shell’s process. The child processes created as a result of interpreting a single command line can be a foreground job or background job. In this lab, I played with signal handlers a lot. For example, handling ended backgroud jobs correctly without interuption the foreground jobs, getting backgroud jobs to foreground, switching job status, etc.

Proxy Lab: Writing a Caching Web Proxy. In this lab I created a sequential web proxy which can get request, parse request header and handle errors. I then extended my proxy to deal with multiple concurrent requests and caching web objects.
